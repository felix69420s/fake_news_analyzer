# fake_news_analyzer

## 1. Название проекта

**fake_news_analyzer** — учебно-исследовательский проект по извлечению аналитических признаков из русскоязычных медиатекстов.

## 2. Цель проекта

Цель проекта — построить воспроизводимый pipeline для извлечения **интерпретируемых признаков** из новостных текстов:

- структурные текстовые признаки;
- признаки именованных сущностей (NER);
- эмоциональный профиль текста;
- rule-based признаки потенциальной манипулятивности.

Проект:

- **не является** полноценной фактчекинг-системой;
- **не выполняет** верификацию фактов по внешним источникам;
- **не предсказывает** класс `fake/real` как итоговое решение.

## 3. Используемые данные

В проекте применяются датасеты из Hugging Face:

- основной: `baiangali/fake_news`;
- дополнительный: `data-silence/lenta.ru_2-extended`.

Датасет Lenta используется без целевой метки `fake/real` и служит в первую очередь для проверки адаптации данных, предобработки и устойчивости пайплайна на реальных медиатекстах.

## 4. Извлекаемые признаки

Pipeline формирует набор признаков по следующим направлениям:

- **NER (Named Entity Recognition):** персоны, организации, локации, геополитические сущности, медиа-сущности;
- **sentiment:** преобладающая тональность и ее score;
- **emotions:** набор эмоциональных меток, их вероятности и доминирующая эмоция;
- **manipulation (rule-based):** эвристические лингвистические маркеры (эмоциональная лексика, страх, обобщения, сенсационность, давление, расплывчатые источники, избыточная пунктуация и др.).

## 5. Структура проекта

Базовая структура:

- `src/config.py` — конфигурация путей, датасетов и моделей;
- `src/schemas.py` — Pydantic-схемы входных и выходных записей;
- `src/dataset_loader.py` — загрузка данных из Hugging Face и локальных файлов;
- `src/dataset_adapters.py` — адаптация датасетов к единому входному формату;
- `src/text_cleaning.py` — очистка и нормализация текста;
- `src/hf_models.py` — ленивая загрузка HF-пайплайнов;
- `src/ner_features.py` — извлечение NER-признаков;
- `src/emotion_features.py` — извлечение sentiment/emotion признаков;
- `src/manipulation_features.py` — rule-based признаки манипулятивности;
- `src/feature_pipeline.py` — обработка одной новости;
- `src/dataset_builder.py` — сборка итогового аналитического датасета;
- `src/main.py` — CLI-точка входа;
- `tests/test_smoke.py` — smoke-тесты без загрузки HF-моделей.

## 6. Установка

1. Создайте и активируйте виртуальное окружение (рекомендуется).
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

## 7. Примеры запуска

Запуск для основного датасета:

```bash
python -m src.main --dataset baiangali --split train --limit 100
```

Запуск для дополнительного датасета Lenta:

```bash
python -m src.main --dataset lenta --split train --limit 100
```

Запуск для локального CSV-файла:

```bash
python -m src.main --dataset local --local-path data/raw/local_news.csv --local-adapter baiangali --limit 100
```

При необходимости можно явно указать устройство и выходной путь:

```bash
python -m src.main --dataset baiangali --device -1 --output data/processed/analytical_dataset.csv
```

## 8. Выходная схема данных

Итоговый аналитический датасет сохраняется в двух форматах:

- CSV (`UTF-8-SIG`) — для удобной работы в Excel;
- JSONL — для сохранения вложенных структур (списки/словари) без потери формата.

Выходная запись включает:

- исходные поля новости (`id`, `date`, `source`, `title`, `text` и др.);
- производные текстовые поля (`full_text`, `normalized_text`, `text_hash`);
- базовые метрики (`char_count`, `token_count_approx`);
- NER-признаки и их агрегаты;
- sentiment/emotion признаки;
- rule-based признаки манипулятивности (`manipulation_flags`, `manipulation_matches`, `manipulation_score`).

## 9. Ограничения

- Rule-based признаки являются **эвристиками** и не эквивалентны доказательству манипуляции.
- Модели NER/sentiment/emotion имеют ограниченную точность и могут ошибаться.
- Полученные признаки не доказывают фейковость текста и должны интерпретироваться в рамках исследовательского анализа, а не как окончательный вердикт.
